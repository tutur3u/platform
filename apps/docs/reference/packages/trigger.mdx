# @tuturuuu/trigger

`@tuturuuu/trigger` is the historical package name for Tuturuuu's reusable
Calendar provider-sync and scheduling helpers. It does not depend on or deploy
to Trigger.dev.

Production recurring work is scheduled by Vercel Cron. The cron route owns
authentication, cadence, observability, and orchestration; this package owns
the testable provider and scheduling operations invoked by that route.

## Calendar provider sync

Use the direct helpers from a server-only route or service:

```ts
import { performFullSyncForWorkspace } from "@tuturuuu/trigger/google-calendar-full-sync";
import { performIncrementalSyncForWorkspace } from "@tuturuuu/trigger/google-calendar-incremental-sync";

await performIncrementalSyncForWorkspace(
  calendarId,
  workspaceId,
  accessToken,
  refreshToken,
);
```

The lower-level `google-calendar-sync` export contains token persistence,
Google event normalization, and batched database synchronization.

## Scheduling helpers

The package also exposes `schedulableTasksHelper` and
`unifiedScheduleHelper`. Call these from authenticated server routes rather
than adding another scheduler:

```ts
import {
  schedulableTasksHelper,
  unifiedScheduleHelper,
} from "@tuturuuu/trigger";

await schedulableTasksHelper(workspaceId);
await unifiedScheduleHelper(workspaceId, {
  windowDays: 30,
  forceReschedule: false,
});
```

## Cron ownership

Calendar provider sync is configured in `apps/web/cron.config.json` and served
through `/api/cron/calendar/provider-sync`. Keep `apps/web/vercel.json`
generated from that configuration with:

```bash
node scripts/sync-web-crons.js
```

Do not add task-runner wrappers, provider-specific recurring jobs, or a second
source of cadence inside this package.

## Validation

```bash
bun run --filter @tuturuuu/trigger test
bun run --filter @tuturuuu/trigger type-check
```
